# User Stories Refinement

Comparación entre las historias de usuario originales y las versiones refinadas, destacando las diferencias clave identificadas durante el análisis.

| HU Original | HU Refinada por la IA | Diferencias Detectadas |
|---|---|---|
| **HU-18:** Como usuario autenticado, quiero verificar la disponibilidad de un espacio en un rango de fecha y hora específico, para saber si puedo crear una reserva en ese horario. | Como usuario autenticado (colaborador), quiero consultar la disponibilidad de un espacio en el rango horario permitido (8:00 a 18:00 UTC), validando solapamientos solo con reservas activas y controlando errores por IDs inválidos, para confirmar si puedo crear la reserva. | Se explicitó la restricción del horario permitido (8:00 a 18:00), se agregó el manejo de errores para datos incorrectos, y se aclaró que los solapamientos se calculan estrictamente sobre reservas activas. |
| **HU-19:** Como usuario autenticado, quiero crear una reserva para un espacio en un horario específico, opcionalmente incluyendo equipos, para asegurar el uso del espacio y los recursos que necesito. | Como usuario autenticado, quiero crear una reserva validando que el horario esté dentro del rango permitido (8:00 a 18:00), pudiendo incluir equipos disponibles en la misma ciudad y gestionando apropiadamente errores y datos opcionales vacíos. | Se añadió la validación explícita del horario permitido, se destacó el formato de fechas y manejo de zonas horarias, y se cubrieron casos de prueba negativos como campos opcionales ausentes o equipos no disponibles. |
| **HU-20:** Como usuario autenticado, quiero consultar la lista de reservas existentes, pudiendo filtrar por usuario, espacio o estado, para revisar las reservas realizadas. | Como usuario autenticado (con permisos de rol), quiero consultar la lista de reservas pertinentes (propias o generales) con filtros combinables, ordenamiento y paginación estructurada, para revisar la información requerida de manera clara. | Se recomendó especificar si el usuario solo puede ver sus propias reservas, se incluyó la necesidad de parámetros de paginación y ordenamiento, aclarando la regla ante filtros combinados inválidos. |
